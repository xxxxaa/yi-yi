# 用户研究与功能细化

## 一、用户画像

### 1.1 核心用户：一线房产经纪人

**基本特征**：
- 年龄：22-40岁
- 学历：大专及以上
- 工作年限：1-10年
- 收入：底薪+提成，月收入波动大

**工作特点**：
- 每天大量时间在外跑盘、带看
- 手机是主要工作工具
- 同时跟进多个客户和多个楼盘
- 业绩压力大，时间碎片化

**技术能力**：
- 熟练使用微信、抖音、小红书
- 会基础的图片编辑（美图秀秀等）
- 不会复杂的视频剪辑
- 对新工具接受度高，但学习时间有限

---

### 1.2 用户细分

#### A. 新手经纪人（入行0-1年）

**特点**：
- 对楼盘不熟悉，需要频繁查资料
- 不知道怎么写文案、拍视频
- 客户少，需要大量获客
- 话术不熟练，容易被客户问住

**核心需求**：
- 快速了解楼盘信息
- 现成的文案模板
- 客户常见问题的标准答案
- 简单易上手的工具

#### B. 成熟经纪人（入行2-5年）

**特点**：
- 有稳定客户来源
- 熟悉区域内楼盘
- 有自己的工作方法
- 追求效率提升

**核心需求**：
- 批量处理重复工作
- 个性化内容生成
- 客户精细化管理
- 数据分析支持决策

#### C. 团队管理者（店长/经理）

**特点**：
- 管理5-20人团队
- 需要培训新人
- 关注团队业绩
- 资源分配决策

**核心需求**：
- 统一的物料管理
- 新人培训支持
- 团队数据看板
- 资源共享机制

---

## 二、用户场景分析

### 2.1 一天的工作流程

```
07:30 - 08:30  早起发朋友圈（房源推荐、早安问候）
08:30 - 09:30  到公司，查看消息，回复客户咨询
09:30 - 10:30  整理今日带看安排，准备资料
10:30 - 12:00  外出带看（可能2-3组客户）
12:00 - 13:30  午餐，发朋友圈，刷抖音找灵感
13:30 - 15:00  继续带看或回访老客户
15:00 - 17:00  跑盘（了解新楼盘，拍素材）
17:00 - 18:30  回公司，录入客户信息，写跟进记录
18:30 - 20:00  晚餐，发朋友圈
20:00 - 22:00  回复客户消息，写小红书/拍视频
22:00 - 23:00  整理明日计划，学习充电
```

### 2.2 关键场景痛点分析

#### 场景1：新楼盘学习

**现状**：
- 开发商给一堆PDF、PPT资料
- 需要自己整理成笔记
- 记不住那么多数据
- 客户问到细节答不上来

**痛点**：
- 资料太多，整理耗时
- 信息分散，查找困难
- 数据更新后要重新整理

**期望**：
- 上传资料自动整理
- 随时能查到任何信息
- 价格更新自动同步

---

#### 场景2：发朋友圈/小红书

**现状**：
- 每天要发3-5条朋友圈
- 不知道写什么内容
- 抄同事的文案没特色
- 拍视频不会剪辑

**痛点**：
- 写文案耗时，质量不高
- 内容同质化严重
- 视频制作门槛高

**期望**：
- 一键生成多条文案
- 每条都有差异化
- 视频脚本直接能用

---

#### 场景3：客户咨询回复

**现状**：
- 客户问楼盘信息，要翻资料
- 问价格要查最新价格表
- 问政策要百度搜索
- 回复慢客户就流失了

**痛点**：
- 信息查找慢
- 回复不够专业
- 容易遗漏重要信息

**期望**：
- 快速查到准确信息
- 有专业话术参考
- 自动提醒重要事项

---

#### 场景4：带看准备

**现状**：
- 要打印户型图、价格表
- 准备楼盘介绍话术
- 规划带看路线
- 经常忘带资料

**痛点**：
- 准备工作繁琐
- 资料版本混乱
- 临时找资料手忙脚乱

**期望**：
- 一键生成带看资料包
- 手机随时能查
- 自动提醒带看事项

---

#### 场景5：客户跟进

**现状**：
- 客户多了记不住需求
- 不知道什么时候该跟进
- 跟进记录写在各种地方
- 容易跟丢客户

**痛点**：
- 客户信息分散
- 跟进时机把握不准
- 缺乏系统化管理

**期望**：
- 统一管理客户信息
- 智能提醒跟进
- 自动记录沟通历史

---

## 三、功能细化设计

### 3.1 楼盘物料管理（重新设计）

#### 用户故事

```
作为一名经纪人，
我希望能快速整理新楼盘资料，
以便随时能查到任何信息回答客户问题。
```

#### 功能清单

| 功能 | 优先级 | 用户价值 |
|------|--------|----------|
| 拍照上传资料 | P0 | 跑盘时直接拍照，不用回来整理 |
| 语音录入信息 | P0 | 边走边说，解放双手 |
| 自动识别整理 | P0 | 省去手动整理时间 |
| 快速搜索查询 | P0 | 客户问什么立刻能答 |
| 价格表更新提醒 | P1 | 避免报错价格 |
| 楼盘对比功能 | P1 | 帮客户做对比分析 |
| 离线查看 | P1 | 没网也能用 |
| 分享给客户 | P2 | 一键发送资料 |

#### 交互设计要点

**上传方式**（按使用频率排序）：
1. 📷 拍照上传 - 最常用，跑盘时拍资料
2. 🎤 语音录入 - 边走边说楼盘信息
3. 📁 文件上传 - 导入开发商给的资料
4. 📋 复制粘贴 - 从其他地方复制文字

**查询方式**：
1. 🔍 自然语言搜索 - "XX楼盘89平户型多少钱"
2. 📂 分类浏览 - 按楼盘/户型/价格筛选
3. ⭐ 常用收藏 - 快速访问常推楼盘

### 3.2 内容生成（重新设计）

#### 用户故事

```
作为一名经纪人，
我希望能快速生成高质量的推广内容，
以便每天发朋友圈和小红书吸引客户。
```

#### 功能清单

| 功能 | 优先级 | 用户价值 |
|------|--------|----------|
| 朋友圈文案生成 | P0 | 每天必用，解决写文案难题 |
| 小红书推文生成 | P0 | 获客主要渠道 |
| 一键多条生成 | P0 | 一次生成一周的内容 |
| 风格切换 | P1 | 避免内容同质化 |
| 短视频脚本 | P1 | 降低视频创作门槛 |
| 历史内容管理 | P1 | 好内容可以复用 |
| 定时发布提醒 | P2 | 提醒最佳发布时间 |
| 效果数据统计 | P2 | 知道什么内容效果好 |

#### 内容类型细化

**朋友圈文案**（每日3-5条）：

| 类型 | 发布时间 | 内容特点 | 示例 |
|------|----------|----------|------|
| 早安问候 | 7:30-8:30 | 正能量+软植入 | "新的一天，愿你离理想的家更近一步🏠" |
| 房源推荐 | 10:00-11:00 | 突出卖点+价格 | "【XX花园】地铁口89平三房，总价298万..." |
| 带看分享 | 14:00-15:00 | 真实感+专业度 | "刚带客户看完XX楼盘，这个户型真的绝了..." |
| 市场资讯 | 17:00-18:00 | 专业形象+价值 | "最新政策解读：首付比例下调意味着..." |
| 成交喜报 | 20:00-21:00 | 信任背书+促单 | "恭喜王姐成功签约XX花园，选房要趁早..." |

**小红书推文**（每周2-3篇）：

| 类型 | 内容结构 | 适用场景 |
|------|----------|----------|
| 探盘笔记 | 第一人称体验分享 | 新盘首发、热门楼盘 |
| 户型分析 | 专业角度解读户型 | 主力户型推广 |
| 避坑指南 | 购房注意事项 | 建立专业形象 |
| 对比测评 | 多楼盘横向对比 | 帮客户做决策 |
| 政策解读 | 最新政策影响分析 | 蹭热点获流量 |

**短视频脚本**：

| 时长 | 适用平台 | 内容结构 |
|------|----------|----------|
| 15秒 | 抖音 | hook+1个卖点+引导 |
| 30秒 | 抖音/视频号 | hook+2-3个卖点+引导 |
| 60秒 | 视频号 | 完整楼盘介绍 |
| 3分钟 | 小红书视频 | 深度探盘vlog |

#### 交互设计要点

**生成流程**（极简化）：
```
选楼盘 → 选类型 → 生成 → 复制使用
   ↓         ↓        ↓
 最近用过   常用类型   一键复制
 智能推荐   快捷入口   直接分享
```

**批量生成**：
- 选择楼盘后，一次生成5条不同风格的朋友圈文案
- 用户可以挑选喜欢的，或者全部保存分批发
