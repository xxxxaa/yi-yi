## 一、产品概述

### 1.1 产品定位

YiYi（易易）是一款面向房产中介从业者的AI智能助手，帮助经纪人高效管理楼盘物料、自动生成营销内容、智能匹配客户需求，提升作业效率。

### 1.2 核心价值

| 价值点 | 描述 |
|--------|------|
| **物料智能化** | 自动解析楼盘资料，结构化存储，随时调用 |
| **内容自动化** | 一键生成短视频脚本、小红书推文、朋友圈文案 |
| **匹配精准化** | 基于客户需求智能推荐房源 |
| **数据本地化** | 客户数据本地存储，安全可控 |

### 1.3 目标用户

**主要用户**：
- 房产经纪人（新房/二手房）
- 房产自媒体运营者
- 房产门店店长

**用户痛点**：
1. 楼盘资料繁多，查找困难
2. 每个楼盘都要手动整理卖点
3. 写推文、拍视频耗时耗力
4. 客户需求记不住，匹配靠感觉

### 1.4 用户画像

#### 新手经纪人（入行0-1年）
- 对楼盘不熟悉，需要频繁查资料
- 不知道怎么写文案、拍视频
- 客户少，需要大量获客
- **核心需求**：快速了解楼盘、现成文案模板、简单易上手

#### 成熟经纪人（入行2-5年）
- 有稳定客户来源，熟悉区域楼盘
- 追求效率提升
- **核心需求**：批量处理、个性化内容、精细化管理

#### 团队管理者（店长/经理）
- 管理5-20人团队
- 需要培训新人，关注团队业绩
- **核心需求**：统一物料管理、新人培训支持、资源共享

### 1.5 典型使用场景

| 场景 | 痛点 | 期望 |
|------|------|------|
| 新楼盘学习 | 资料太多整理耗时，记不住数据 | 上传自动整理，随时能查 |
| 发朋友圈 | 每天要发3-5条，不知道写什么 | 一键生成多条差异化文案 |
| 客户咨询 | 翻资料慢，回复不够专业 | 快速查到准确信息 |
| 带看准备 | 准备工作繁琐，资料版本混乱 | 一键生成带看资料包 |
| 客户跟进 | 客户多记不住，容易跟丢 | 统一管理，智能提醒 |

## 三、楼盘物料管理中心

### 3.1 功能概述

为每个楼盘建立独立的物料库，支持多种格式物料的上传、解析、存储和检索。

### 3.2 物料类型支持

| 物料类型 | 格式 | 解析能力 |
|----------|------|----------|
| 楼盘资料 | PDF/Word/PPT | 提取文字、表格、图片 |
| 户型图 | JPG/PNG | OCR识别面积、朝向、房型 |
| 效果图 | JPG/PNG | 场景识别、标签提取 |
| 视频素材 | MP4/MOV | 关键帧提取、语音转文字 |
| 价格表 | Excel/图片 | 表格解析、价格提取 |
| 区位图 | 图片 | 周边配套识别 |
| 销售说辞 | Word/PDF/音频 | 卖点提取、话术整理 |

### 3.3 物料解析流程

```
上传物料 → 格式识别 → 内容提取 → 结构化存储 → 知识索引
    │           │           │            │           │
    ▼           ▼           ▼            ▼           ▼
  文件存储   类型分类    AI解析      数据入库    向量化
```

### 3.4 楼盘信息结构

```yaml
楼盘基础信息:
  - 楼盘名称
  - 开发商
  - 物业公司
  - 地址/区位
  - 交房时间
  - 产权年限

产品信息:
  - 建筑类型（高层/洋房/别墅）
  - 梯户比
  - 户型列表
    - 户型名称
    - 面积
    - 朝向
    - 售价/单价
    - 户型图
    - 卖点描述

配套信息:
  - 交通（地铁/公交/高速）
  - 教育（学校/学区）
  - 商业（商场/超市）
  - 医疗（医院/诊所）
  - 公园/休闲

卖点提炼:
  - 核心卖点（3-5个）
  - 差异化优势
  - 目标客群
  - 竞品对比

物料资源:
  - 效果图列表
  - 户型图列表
  - 视频素材
  - 销售说辞
  - 价格表
```

### 3.5 物料管理功能

| 功能 | 描述 |
|------|------|
| 批量上传 | 支持拖拽上传多个文件 |
| 自动分类 | AI识别物料类型自动归类 |
| 智能解析 | 提取关键信息结构化存储 |
| 版本管理 | 价格表等物料支持版本更新 |
| 快速检索 | 自然语言查询楼盘信息 |
| 物料预览 | 在线预览各类物料 |
| 导出分享 | 导出物料包或分享链接 |

## 四、内容生成引擎

### 4.1 功能概述

基于楼盘物料库，一键生成多种营销内容，支持多平台适配。

### 4.2 支持的内容类型

#### 4.2.1 短视频脚本

```yaml
输出格式:
  - 视频时长: 15s/30s/60s/3min
  - 脚本结构:
      - 开场hook（吸引注意）
      - 楼盘介绍（核心卖点）
      - 户型展示（面积价格）
      - 配套说明（交通教育）
      - 结尾引导（联系方式）
  - 配套输出:
      - 分镜建议
      - 画面描述
      - 字幕文案
      - BGM建议
      - 拍摄地点建议

视频类型:
  - 楼盘整体介绍
  - 单户型详解
  - 样板间实拍
  - 周边配套探访
  - 价格解读
  - 对比测评
```

#### 4.2.2 小红书推文

```yaml
输出格式:
  - 标题: 吸睛标题（含emoji）
  - 正文: 800-1500字种草文案
  - 标签: 相关话题标签
  - 图片建议: 封面图+内容图顺序

内容风格:
  - 探盘笔记风
  - 干货分享风
  - 避坑指南风
  - 对比测评风
  - 购房攻略风

示例模板:
  标题: "🏠上海刚需福音！总价300万上车地铁房"
  正文: |
    姐妹们！最近踩盘发现一个宝藏楼盘...
    📍位置：xxx
    🏷️价格：xxx
    ✨亮点：xxx
    ⚠️注意：xxx
  标签: #上海买房 #刚需上车 #地铁房
```

#### 4.2.3 朋友圈文案

```yaml
输出格式:
  - 文案长度: 50-200字
  - 配图建议: 1-9张图片顺序
  - 发布时间建议

文案类型:
  - 新盘首发
  - 户型推荐
  - 价格播报
  - 成交喜报
  - 活动预告
  - 政策解读
```

#### 4.2.4 其他内容

| 内容类型 | 描述 |
|----------|------|
| 抖音文案 | 适配抖音风格的短文案 |
| 视频号文案 | 微信视频号适配内容 |
| 公众号文章 | 长文深度分析 |
| 海报文案 | 配合设计的文案内容 |
| 客户话术 | 针对不同客户的沟通话术 |

### 4.3 内容生成流程

```
选择楼盘 → 选择内容类型 → 选择风格/模板 → AI生成 → 人工调整 → 导出发布
    │           │              │            │          │          │
    ▼           ▼              ▼            ▼          ▼          ▼
 物料调用    参数配置       风格适配      内容输出    编辑优化    多平台
```

### 4.4 生成参数配置

| 参数 | 选项 |
|------|------|
| 目标平台 | 抖音/小红书/朋友圈/视频号/公众号 |
| 内容风格 | 专业/轻松/种草/干货/情感 |
| 目标客群 | 刚需/改善/投资/养老 |
| 突出卖点 | 可选择强调的卖点 |
| 内容长度 | 短/中/长 |
| 是否含价格 | 是/否 |

## 六、智能匹配引擎

## 九、数据模型

### 9.1 楼盘表 (projects)

```sql
CREATE TABLE projects (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,           -- 楼盘名称
  developer TEXT,               -- 开发商
  address TEXT,                 -- 地址
  city TEXT,                    -- 城市
  district TEXT,                -- 区域
  status TEXT,                  -- 状态：在售/待售/售罄
  property_type TEXT,           -- 物业类型
  property_years INTEGER,       -- 产权年限
  delivery_date TEXT,           -- 交房时间
  created_at DATETIME,
  updated_at DATETIME
);
```

### 9.2 户型表 (house_types)

```sql
CREATE TABLE house_types (
  id TEXT PRIMARY KEY,
  project_id TEXT,              -- 关联楼盘
  name TEXT,                    -- 户型名称
  rooms INTEGER,                -- 室
  halls INTEGER,                -- 厅
  bathrooms INTEGER,            -- 卫
  area_min REAL,                -- 最小面积
  area_max REAL,                -- 最大面积
  price_total_min REAL,         -- 最低总价
  price_total_max REAL,         -- 最高总价
  price_unit REAL,              -- 单价
  orientation TEXT,             -- 朝向
  features TEXT,                -- 特点JSON
  FOREIGN KEY (project_id) REFERENCES projects(id)
);
```

### 9.3 物料表 (materials)

```sql
CREATE TABLE materials (
  id TEXT PRIMARY KEY,
  project_id TEXT,              -- 关联楼盘
  type TEXT,                    -- 类型：image/video/document/audio
  category TEXT,                -- 分类：效果图/户型图/价格表等
  file_path TEXT,               -- 文件路径
  file_name TEXT,               -- 文件名
  file_size INTEGER,            -- 文件大小
  mime_type TEXT,               -- MIME类型
  parsed_content TEXT,          -- 解析后的内容JSON
  version INTEGER DEFAULT 1,    -- 版本号
  created_at DATETIME,
  updated_at DATETIME,
  FOREIGN KEY (project_id) REFERENCES projects(id)
);
```

### 9.4 客户表 (customers)

```sql
CREATE TABLE customers (
  id TEXT PRIMARY KEY,
  name TEXT,                    -- 姓名
  phone TEXT,                   -- 电话
  wechat TEXT,                  -- 微信
  source TEXT,                  -- 来源
  level TEXT,                   -- 等级：A/B/C/D
  stage TEXT,                   -- 阶段
  budget_min REAL,              -- 预算下限
  budget_max REAL,              -- 预算上限
  area_min REAL,                -- 面积下限
  area_max REAL,                -- 面积上限
  preferred_districts TEXT,     -- 偏好区域JSON
  requirements TEXT,            -- 需求详情JSON
  created_at DATETIME,
  updated_at DATETIME
);
```

### 9.5 生成内容表 (generated_contents)

```sql
CREATE TABLE generated_contents (
  id TEXT PRIMARY KEY,
  project_id TEXT,              -- 关联楼盘
  type TEXT,                    -- 类型：video_script/xiaohongshu/moments
  platform TEXT,                -- 平台
  title TEXT,                   -- 标题
  content TEXT,                 -- 内容
  params TEXT,                  -- 生成参数JSON
  created_at DATETIME,
  FOREIGN KEY (project_id) REFERENCES projects(id)
);
```

## 十、用户流程

### 10.1 楼盘物料管理流程

```
1. 创建楼盘
   └→ 输入楼盘基础信息（名称、地址、开发商等）

2. 上传物料
   └→ 批量上传PDF/图片/视频等资料
   └→ 系统自动识别物料类型

3. 物料解析
   └→ AI自动解析物料内容
   └→ 提取关键信息（户型、价格、配套等）
   └→ 用户确认/修正解析结果

4. 信息完善
   └→ 补充/修改楼盘信息
   └→ 添加卖点描述
   └→ 设置目标客群

5. 物料管理
   └→ 查看/预览物料
   └→ 更新物料版本
   └→ 删除过期物料
```

### 10.2 内容生成流程

```
1. 选择楼盘
   └→ 从楼盘列表选择目标楼盘

2. 选择内容类型
   └→ 短视频脚本 / 小红书推文 / 朋友圈文案

3. 配置参数
   └→ 选择风格/模板
   └→ 选择突出卖点
   └→ 选择目标客群

4. AI生成
   └→ 基于物料库生成内容
   └→ 展示生成结果

5. 编辑调整
   └→ 人工修改优化
   └→ 重新生成（可选）

6. 导出使用
   └→ 复制文案
   └→ 导出脚本文档
   └→ 保存到素材库
```

### 10.3 客户匹配流程

```
1. 录入客户
   └→ 手动输入 / 对话提取

2. 需求分析
   └→ AI分析客户需求
   └→ 生成需求画像

3. 智能匹配
   └→ 匹配合适楼盘
   └→ 展示匹配结果和原因

4. 生成话术
   └→ 针对客户生成推荐话术
   └→ 准备带看资料
```

## 十一、用户体验原则

### 11.1 设计原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **移动优先** | 主要在手机上使用 | 大按钮、简洁界面 |
| **一步到位** | 减少操作步骤 | 一键生成、一键复制 |
| **离线可用** | 外出时网络不稳定 | 本地缓存、离线查看 |
| **快速响应** | 客户在等回复 | 秒级响应、流式输出 |
| **容错友好** | 用户可能操作失误 | 撤销、重新生成 |

### 11.2 关键交互设计

**首页布局**：
- 今日待办提醒（待跟进客户数）
- 三大快捷入口：写文案、查楼盘、看客户
- 对话输入框（支持自然语言指令）
- 快捷操作按钮

**文案生成流程**：
```
选楼盘（最近使用优先）→ 选类型（默认最常用）→ 一键生成 → 复制使用
```
