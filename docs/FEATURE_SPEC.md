# 功能规格说明书

## 一、朋友圈文案生成

### 1.1 功能概述

帮助经纪人快速生成高质量的朋友圈推广文案，支持多种类型和风格。

### 1.2 用户流程

```
┌─────────────────────────────────────────────────────────────┐
│                        入口                                  │
│  首页快捷入口 / 楼盘详情页 / 对话指令                        │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                     选择楼盘                                 │
│  - 搜索楼盘名称                                             │
│  - 最近使用的楼盘（最多显示5个）                            │
│  - 收藏的楼盘                                               │
│  【如果只有1个楼盘，自动选中】                              │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                     选择文案类型                             │
│  ○ 房源推荐（默认选中）                                     │
│  ○ 带看分享                                                 │
│  ○ 成交喜报                                                 │
│  ○ 早安问候                                                 │
│  ○ 市场资讯                                                 │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                     高级选项（可选）                         │
│  - 突出卖点：[地铁] [学区] [价格] [户型] ...               │
│  - 生成数量：1/3/5条                                        │
│  - 是否包含价格：是/否                                      │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                     生成结果                                 │
│  显示生成的文案列表，每条文案包含：                         │
│  - 文案内容                                                 │
│  - 配图建议                                                 │
│  - 操作按钮：[复制] [重新生成] [保存]                       │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 文案类型规格

#### 房源推荐

**输入要求**：
- 必须选择楼盘
- 可选择具体户型

**输出规格**：
```
长度：80-150字
结构：
  - 楼盘名称+定位（1句）
  - 核心卖点（2-3个）
  - 户型+价格信息
  - 行动号召（可选）

示例：
【XX花园】浦东核心地段，地铁2号线上盖
✅ 89㎡南北通透三房，主卧带飘窗
✅ 自带12年一贯制学校
✅ 精装交付，拎包入住
总价298万起，首付90万上车
看房私聊，周末有专车接送🚗
```

**配图建议**：
- 第1张：效果图或实景图
- 第2-3张：户型图
- 第4-6张：样板间图
- 第7-9张：配套/区位图

#### 带看分享

**输入要求**：
- 必须选择楼盘
- 可输入带看感受（可选）

**输出规格**：
```
长度：100-200字
结构：
  - 带看引入（今天/刚刚带客户看了...）
  - 实地感受（真实体验描述）
  - 亮点发现（1-2个）
  - 适合人群
  - 软性引导

示例：
刚带客户看完XX花园的样板间，说实话被惊艳到了👀

89㎡做到了三房两卫，空间利用率真的高！
客厅开间3.9米，落地窗采光超棒☀️
最惊喜的是主卧，带独立衣帽间+飘窗，住着太舒服了

这个户型特别适合三口之家，预算300万左右的可以重点考虑
想看的私信我，周末还有几个名额📝
```

#### 成交喜报

**输入要求**：
- 必须选择楼盘
- 可输入客户称呼（可选，默认用"客户"）
- 可输入成交户型（可选）

**输出规格**：
```
长度：50-100字
结构：
  - 恭喜开头
  - 成交信息（楼盘+户型）
  - 感谢/祝福
  - 软性促单

示例：
🎉恭喜张姐成功签约XX花园89㎡三房！

从第一次看房到签约只用了2周
果断的人运气都不会差💪

感谢信任，祝早日拿到新房钥匙🔑
还在观望的朋友，好房不等人哦~
```

#### 早安问候

**输入要求**：
- 楼盘可选（不选则生成通用问候）

**输出规格**：
```
长度：30-80字
结构：
  - 问候语
  - 正能量/生活感悟
  - 软性植入（可选）

示例：
早安☀️

新的一天，愿你离理想的家更近一步🏠
无论是寻找第一个家，还是换一个更好的家
都值得被认真对待

有任何买房问题，随时找我聊~
```

### 1.4 边界情况处理

| 情况 | 处理方式 |
|------|----------|
| 楼盘信息不完整 | 提示"该楼盘信息不完整，建议先补充资料" |
| 没有楼盘数据 | 引导用户先添加楼盘 |
| 生成失败 | 显示"生成失败，请重试"，提供重试按钮 |
| 网络断开 | 提示"网络不可用，请检查网络连接" |
| 内容包含敏感词 | 自动过滤后输出，不提示用户 |

### 1.5 数据存储

```typescript
interface GeneratedMoments {
  id: string;
  projectId: string;
  type: 'recommend' | 'visit' | 'deal' | 'greeting' | 'news';
  content: string;
  imagesSuggestion: string[];
  createdAt: Date;
  usedAt?: Date;  // 用户复制时记录
}
```

## 二、小红书推文生成

### 2.1 功能概述

生成适合小红书平台的种草笔记，包含标题、正文、标签和配图建议。

### 2.2 用户流程

```
选择楼盘 → 选择笔记类型 → 配置选项 → 生成 → 编辑调整 → 复制使用
```

### 2.3 笔记类型规格

#### 探盘笔记

**输出规格**：
```yaml
标题：
  长度：15-20字
  格式：emoji + 吸睛标题
  示例："🏠上海刚需福音！总价300万上车浦东地铁房"

正文：
  长度：800-1200字
  结构：
    - 开场（50-100字）：引入话题，制造好奇
    - 楼盘介绍（150-200字）：位置、开发商、基本信息
    - 户型分析（200-300字）：主推户型详细介绍
    - 配套说明（150-200字）：交通、学校、商业
    - 价格分析（100-150字）：价格区间、性价比
    - 总结建议（50-100字）：适合人群、购买建议

标签：
  数量：8-12个
  分类：
    - 城市标签：#上海买房 #上海新房
    - 区域标签：#浦东买房 #张江
    - 需求标签：#刚需上车 #首套房
    - 产品标签：#地铁房 #学区房
    - 通用标签：#买房攻略 #房产干货

配图建议：
  封面：效果图或实景图（竖版更佳）
  内容图：户型图、样板间、配套实景
  数量：6-9张
```

#### 户型分析

**输出规格**：
```yaml
标题：
  示例："89㎡做出三房两卫？这个户型设计绝了！"

正文：
  长度：600-1000字
  结构：
    - 户型概览（基本参数）
    - 空间分析（逐个房间分析）
    - 优点总结（3-5个亮点）
    - 缺点提醒（1-2个注意点）
    - 适合人群
    - 购买建议
```

#### 避坑指南

**输出规格**：
```yaml
标题：
  示例："买房前必看！这5个坑千万别踩"

正文：
  长度：800-1500字
  结构：
    - 引入（为什么写这篇）
    - 坑点列表（每个坑点详细说明）
    - 如何避坑（实用建议）
    - 总结
```

### 2.4 风格选项

| 风格 | 特点 | 适用场景 |
|------|------|----------|
| 专业分析 | 数据详实、逻辑清晰 | 高端楼盘、投资客 |
| 轻松种草 | 口语化、有趣味 | 刚需楼盘、年轻人 |
| 真实体验 | 第一人称、有细节 | 探盘笔记 |
| 干货分享 | 信息密度高、实用 | 攻略类内容 |

### 2.5 输出格式

```typescript
interface XiaohongshuPost {
  title: string;
  content: string;
  tags: string[];
  coverSuggestion: string;
  imagesSuggestion: string[];
  publishTimeSuggestion: string;  // 建议发布时间
}
```

## 三、楼盘信息查询

### 3.1 功能概述

通过自然语言查询楼盘相关信息，快速获得准确答案。

### 3.2 查询类型

#### 基础信息查询

**示例问题**：
- "XX花园在哪里"
- "XX花园是哪个开发商"
- "XX花园什么时候交房"

**回答格式**：
```
XX花园位于浦东新区张江镇xxx路xxx号，
距离地铁2号线张江高科站约800米步行距离。
```

#### 户型查询

**示例问题**：
- "XX花园有哪些户型"
- "XX花园89平的户型怎么样"
- "XX花园最小的户型多大"

**回答格式**：
```
XX花园89㎡户型（A户型）：

📐 基本信息
- 3室2厅1卫，南北通透
- 建面89㎡，套内约72㎡

💰 价格信息
- 总价：298-320万
- 单价：约3.4万/㎡

✨ 户型亮点
- 南北通透，采光通风好
- 主卧带飘窗，约15㎡
- 客厅开间3.9米

⚠️ 注意事项
- 北向次卧较小，约8㎡
```

#### 价格查询

**示例问题**：
- "XX花园多少钱"
- "XX花园89平总价多少"
- "XX花园单价多少"

**回答格式**：
```
XX花园价格信息（更新于2026-02-01）：

户型        面积      总价        单价
A户型      89㎡     298-320万   3.3-3.6万

💡 目前在售房源以中高楼层为主，低楼层基本售罄
```

#### 配套查询

**示例问题**：
- "XX花园附近有地铁吗"
- "XX花园学区是哪个学校"
- "XX花园周边有什么商场"

**回答格式**：
```
XX花园周边配套：

🚇 交通
- 地铁2号线张江高科站（800米）
- 公交xxx路、xxx路

🏫 教育

- 中学：xxx中学（1.2公里）

🛒 商业
- xxx商场（1公里）
- xxx超市（300米）

🏥 医疗
- xxx医院（2公里）
```

#### 对比查询

**示例问题**：
- "XX花园和YY公馆哪个好"
- "对比一下XX花园和YY公馆"

**回答格式**：
```
XX花园 vs YY公馆 对比：

          XX花园          YY公馆
位置      浦东张江        浦东金桥
开发商    xxx地产         yyy地产

交房      2027年6月       2026年12月

💡 总结建议：
- 追求性价比选XX花园
- 着急入住选YY公馆
- 看重学区需要具体了解对口政策
```

### 3.3 查询失败处理

| 情况 | 回复 |
|------|------|
| 楼盘不存在 | "没有找到'XXX'的信息，要帮你添加这个楼盘吗？" |
| 信息不完整 | "XX花园的价格信息还没有录入，要现在补充吗？" |
| 问题不明确 | "你想了解XX花园的哪方面信息？比如价格、户型、配套..." |

## 四、楼盘物料上传

### 4.1 功能概述

支持多种方式上传楼盘资料，自动解析提取关键信息。

### 4.2 上传方式

#### 拍照上传

**流程**：
```
点击拍照 → 拍摄资料 → 确认图片 → 选择楼盘 → 选择类型 → 上传解析
```

**支持的资料类型**：
- 户型图
- 价格表
- 楼盘资料页
- 区位图
- 宣传单页

**解析能力**：
- OCR识别文字
- 表格结构识别
- 关键信息提取

#### 文件上传

**支持格式**：
| 格式 | 最大大小 | 说明 |
|------|----------|------|
| PDF | 50MB | 楼书、户型册 |
| Word | 20MB | 销售说辞、资料 |
| Excel | 10MB | 价格表、房源表 |
| PPT | 50MB | 楼盘介绍 |
| 图片 | 10MB/张 | JPG/PNG |
| 视频 | 200MB | MP4/MOV |

#### 语音录入

**流程**：
```
点击录音 → 说出信息 → 停止录音 → AI整理 → 确认保存
```

**示例**：
```
用户说："XX花园，开发商是xxx地产，位置在浦东张江，
主力户型89平三房，总价300万左右，地铁2号线800米，
对口xxx小学"

AI整理：
楼盘名称：XX花园
开发商：xxx地产
位置：浦东张江
主力户型：89㎡三房
总价：约300万
交通：地铁2号线800米
学区：xxx小学

确认保存？[确认] [修改]
```

### 4.3 解析结果确认

上传后显示解析结果，用户可以：
- 确认正确 → 直接保存
- 部分修改 → 编辑后保存
- 完全错误 → 手动录入

### 4.4 物料版本管理

**价格表更新场景**：
```
检测到已有XX花园的价格表（上传于2026-01-15）
新上传的价格表将：
○ 替换旧版本（推荐）
○ 保留两个版本
```

## 五、客户快速录入

快速记录新客户信息，支持多种录入方式。

### 5.2 录入方式

**示例**：
```
用户：新客户张先生，预算300万，想买浦东三房

AI：已创建客户档案：
┌─────────────────────────┐
│ 👤 张先生               │

│ 📍 区域：浦东           │

- 联系电话
- 购房目的（自住/投资）
```

| 字段 | 必填 | 类型 | 说明 |

- 根据区域：浦东客/浦西客

**手动标签**：
- 用户可自定义标签

### 5.4 客户状态流转

```
新客户 → 已沟通 → 已带看 → 谈判中 → 已成交
```

## 七、费用计算器

### 7.1 功能概述

快速计算购房相关费用，包括税费、月供、首付等。

### 7.2 计算类型

#### 月供计算

**输入**：
- 贷款金额
- 贷款年限（5/10/15/20/25/30年）
- 利率（默认当前LPR）
- 还款方式（等额本息/等额本金）

**输出**：
```
贷款金额：200万
贷款年限：30年
利率：4.2%（LPR）
还款方式：等额本息

月供：9,778元
总利息：152万
总还款：352万
```

#### 税费计算

**输入**：
- 房屋总价
- 房屋面积
- 是否首套
- 是否满五唯一（二手房）
- 买方是否首次购房

**输出**：
```
房屋总价：300万
面积：89㎡
首套房：是

契税：3万（1%）
中介费：6万（2%）
其他费用：约0.5万

总费用：约9.5万
```

#### 首付计算

**输入**：
- 房屋总价
- 首付比例（首套/二套）
- 公积金可用额度

**输出**：
```
房屋总价：300万
首付比例：30%（首套）

首付金额：90万
贷款金额：210万
月供（30年）：约10,266元
```

### 7.3 快捷入口

对话中可直接触发：
- "帮我算一下300万的房子月供多少"
- "首付90万能买多少钱的房子"
- "300万的房子要交多少税"

## 八、3S快速讲房

### 8.1 功能概述

3秒内快速介绍楼盘核心卖点，适用于电梯演讲、快速推荐、电话沟通等场景。

### 8.2 用户流程

```
选择楼盘 → 选择卖点侧重 → 一键生成 → 复制/朗读
```

### 8.3 话术结构

```yaml
3S话术模板:
  结构: "[楼盘定位]，[核心卖点]，[价格/行动引导]"

  示例1（地铁优势）:
    "XX花园，地铁口300米纯新盘，89平三房总价280万起，首付50万上车！"

  示例2（学区优势）:
    "XX花园，对口市重点xx小学，89平三房298万，学区房里性价比最高！"

  示例3（价格优势）:
    "XX花园，板块内单价最低的新盘，89平三房总价280万，比周边便宜20%！"

  示例4（品质优势）:
    "XX花园，xxx地产品质大盘，精装交付带地暖，89平三房320万起！"
```

### 8.4 生成参数

| 参数 | 选项 | 说明 |
|------|------|------|
| 卖点侧重 | 位置/价格/学区/品质/户型 | 选择最突出的卖点 |
| 目标客群 | 刚需/改善/投资 | 影响话术风格 |
| 语气风格 | 专业/亲切/紧迫 | 影响表达方式 |
| 是否含价格 | 是/否 | 是否显示具体价格 |

### 8.5 输出格式

```typescript
interface ThreeSecondPitch {
  content: string;           // 话术内容
  emphasis: string;          // 卖点侧重
  targetAudience: string;    // 目标客群
  audioUrl?: string;         // 语音版本（可选）
}
```

### 9.3 讲解场景

```yaml

  - 项目规划（1-2分钟）
    "接下来我们看项目的整体规划..."
    - 总占地、总建面
    - 容积率、绑化率
    - 楼栋分布
    - 配套设施（会所、幼儿园等）

  - 产品介绍（1分钟）
    "项目主要有以下几种户型..."
    - 户型概览
    - 主力户型推荐

  - 过渡引导
    "接下来我带您去看看样板间..."
```

#### 样板间讲解

```yaml
话术结构:
  - 入户玄关
    "我们现在进入的是89㎡三房户型..."

  - 客餐厅
    "客厅开间3.9米，采光面很大..."
    - 面积、开间
    - 采光、通风
    - 装修亮点

  - 主卧
    "主卧朝南，带独立卫生间..."
    - 面积、朝向
    - 功能区划分
    - 收纳空间

    "这间次卧可以做儿童房或书房..."

  - 厨卫

  - 阳台
    "阳台面积约6㎡，可以做休闲区..."

  - 总结
    "这个户型最大的亮点是..."
```
